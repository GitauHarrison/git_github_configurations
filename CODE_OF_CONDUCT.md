# Code of Conduct for the Git & GitHub Configurations Helper

This project helps users automate and standardize their Git and GitHub setup. We want the community around it to be constructive, respectful, and inclusive.

## Expectations

We expect everyone participating in this project to:

- Be respectful and considerate in all interactions.
- Assume good intentions and ask clarifying questions when needed.
- Provide constructive feedback and avoid personal attacks.
- Use inclusive language and consider different levels of Git/GitHub experience.

## Unacceptable behavior

Unacceptable behaviors include, but are not limited to:

- Harassment, discrimination, or derogatory comments.
- Threats of violence or encouragement of self-harm.
- Repeated unwanted contact or attention.
- Publishing others' private information (e.g. personal email, SSH key material) without consent.
- Any conduct that would be considered inappropriate in a professional environment.

## Scope

This Code of Conduct applies to all interactions in this repository (issues, pull requests, reviews, comments) and to any associated communication channels for this project.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please contact the project maintainer using the contact information in the README or project metadata.

When reporting, include:

- What happened.
- Where it happened (issue, PR, commit, etc.).
- When it happened.
- Any additional context you think is important.

## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct. They may take any action they deem appropriate, including but not limited to:

- A private warning.
- Temporary or permanent restriction from contributing to this project.

By participating in this project, you agree to abide by this Code of Conduct and help keep the community around these automation tools supportive and welcoming.
